# 成熟游戏差距清单（按当前实现校准）

> 这份清单不是 Steam 发售清单，而是“距离一个结构稳定、体验连续、能继续长期迭代的游戏”还差什么。
> 本版本以当前代码和资源现状为准，不再回退到旧的 `runtime builder 直拼 UI` 思路。

状态说明：

- `[x]` 已完成，当前实现可作为后续基线
- `[-]` 已部分完成，方向正确，但还需要补验证、资源落盘或打磨
- `[ ]` 尚未完成，仍是明确缺口

## 当前基线

- [x] UI 主流程已经统一到 `AppRoot + GlobalGameFlowManager + FSMKit`，不再是主菜单/大地图/历练各自散写主流程。
- [x] 项目级 UI 调度已经统一到 `GameUiPanelId + GameUiPanelRegistry + IGameUiService`。
- [x] 全屏主界面互斥规则已经在代码层落地：`UILevel.Bg` 下的排他面板会自动隐藏同层其他主界面。
- [x] `prefab-first` 已经是正式运行时路径，builder 主要保留为编辑器导出工具。
- [x] `Tooltip / MessagePopup / GameHub / PlayerCompendium / WorldMap* / Expedition` 都已经接入统一面板体系。
- [x] NPC 对话系统、敌人 AI intent 预览、任务/剧情摘要已经接入世界地图与战斗主链路。
- [x] 世界地图 UI 资源导出链路已经补齐，`GameHub / PlayerCompendium / WorldMapNpcDialogue` 等 prefab 已按当前注册表重新落盘。

## A. UI 硬规范

### UI-01. 全屏主界面唯一显示

- [x] 现有正式主界面已经按 `GameUiPanelId` 收口，`MainMenu / WorldMap / WorldMapRegion / WorldMapSettlement / WorldMapSectResidence / Expedition` 走统一打开路径。
- [x] 同层排他规则已在 `GameUiService.ClosePanelsAtSameLevel(...)` 落地，不再依赖调用顺序“碰运气”避免重叠。
- [-] 还需要补一轮实际场景回归，确认没有漏网入口绕开统一调度。

### UI-02. 只有弹窗允许覆盖主界面

- [x] `MessagePopup`、`Tooltip`、主菜单设置/读档/建档等都已经与主界面分层。
- [x] 世界地图背包、工坊、修士总览、NPC 对话等覆盖层都已经拆成独立面板，而不是继续塞在同一个 root 里切块显隐。
- [-] “统一 ModalBlocker 规范” 目前是按面板 prefab 各自带遮罩实现，不是单独的全局阻断层资产。

### UI-03. 统一 UI 状态机 / UI Root

- [x] `AppRoot` 已承接全局流程状态，`CultivationUIPanel / CultivationController` 已成为表现层统一入口。
- [x] 主菜单和历练都已经切到 `FSMKit` 管理主要状态切换。
- [-] `Esc / 返回键` 行为已经在多个主界面里显式处理，但还没有沉淀成一份项目级输入规范文档，也缺少完整自动化覆盖。

### UI-04. 明确分层

- [x] 当前已明确使用 `UILevel.Bg / PopUI / Const / AlwayTop`。
- [x] `MainMenu / WorldMap / Expedition` 等全屏页在 `Bg`，`MessagePopup` 在 `PopUI`，`Tooltip` 在顶层，`GameHub` 在常驻层。
- [-] 当前层级已经足够支撑运行，但还没有把 `ToastLayer / BlockerLayer` 概念抽成独立命名资源和统一样式约束。

### UI-05. 从源头减少重叠

- [x] 核心主界面已切回静态 prefab，布局不再主要依赖运行时代码拼装。
- [x] `WorldMapNpcDialoguePanel` 等新面板已经补上按窗口尺寸缩放的响应式逻辑。
- [ ] 还没有完成系统性的多分辨率冒烟验证，`1280x720 / 1920x1080 / 21:9` 仍需逐页实跑检查。

## B. 成熟 UI 还需补强

### UI-06. 导航一致性

- [x] 主菜单、世界地图、历练内暂停都已有明确返回路径。
- [x] 有阻塞面板时优先关闭面板、无阻塞面板时执行当前主流程返回，这个方向已经形成。
- [-] 手柄/键盘焦点流没有正式收口，后续如果做手柄支持，仍需要补一轮导航规则。

### UI-07. 信息密度与阅读层次

- [-] 大地图、修士总览、门派驻地、NPC 对话已经开始区分“摘要区 / 操作区 / 详情区”，比旧版散点 UI 明显进步。
- [ ] 还没有形成项目级的信息层级规范，部分页面仍然偏“功能先堆上去，再慢慢整理”。

### UI-08. 布局系统化

- [x] 现在主路径是静态 prefab + TMP，不再继续扩大“运行时代码直接拼布局”的覆盖面。
- [x] 世界地图子面板、Overlay 面板、主菜单设置等都已经拆分成独立 prefab/controller。
- [ ] 还没有正式的 design token 体系，字号、边距、按钮尺寸、强调色仍主要体现在 builder 和 prefab 具体实现里。

### UI-09. 视觉反馈完整

- [x] 关键操作已经有 `ShowSuccessMessage / ShowWarningMessage / ShowErrorMessage` 与状态文案反馈。
- [x] 按钮音效、Tooltip、战斗反馈音、命中/受击/治疗表现资源已经接上正式链路。
- [-] hover/disabled/selected 的统一视觉规范还不完整，不同面板之间的一致性需要继续收敛。

## C. 体验成熟度

### UX-01. 新手引导

- [ ] 还没有真正的新手引导流程。
- [-] 当前只有主菜单提示、世界地图提示、任务摘要、NPC 线索提示，能辅助理解，但还不构成首次游玩的完整 onboarding。

### UX-02. 主循环表达

- [-] 现有主循环已经不只是“切场景”：世界地图、地域推进、任务委托、宗门事务、NPC 对话、历练战斗之间已经有因果链。
- [-] 但“当前最值得做的下一件事”仍需要更强的前置提示和阶段目标表达。

### UX-03. 设置与基础交互

- [x] 已有音乐 / 音效 / 语音三通道音量和全屏切换。
- [ ] 分辨率、窗口模式细分、画质基础项、输入说明等仍未补齐。
- [ ] 退出确认、自动保存提示、更多异常状态提示仍需完善。

### UX-04. 失败恢复与异常体验

- [-] 缺 prefab、场景未进 Build Settings 等常见问题，当前已有玩家可见错误提示，而不是只打一条控制台日志。
- [ ] 存档损坏、加载失败、资源缺失后的恢复策略还不完整，仍需要正式的兜底流程。

## D. 内容成熟度

### CONTENT-01. 核心内容量

- [-] 已经不是“只有空壳系统”：任务、剧情摘要、门派驻地、城镇整备、NPC 对话、地域线索、敌人 AI 都已具备基本内容骨架。
- [ ] 但内容量还不足以支撑长时间重复游玩，很多玩法仍在“结构已搭好，量和差异度不够”的阶段。

### CONTENT-02. 数值与节奏

- [-] 战斗、奖励、成长、任务推进已经有正式系统，不再是纯占位。
- [ ] 前 10 分钟 / 30 分钟 / 1 小时体验节奏还没有定稿，数值平衡也没有进入稳定打磨阶段。

### CONTENT-03. 中长期目标

- [-] 已经开始出现中期目标雏形：任务推进、门派事务、角色成长、修仙技艺分支、地域探索。
- [ ] 长期身份、结局导向、世界变化感仍不够强，需要更多明确的阶段性里程碑。
